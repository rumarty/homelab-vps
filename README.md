# VPS Infrastructure as Code (Ansible)

Репозиторий содержит IaC (Infrastructure as Code) для автоматического развертывания и конфигурирования VPS-сервера на базе **Ubuntu 22.04**. 

## Что внутри
* **Role: common** — Обновление системы, установка базовых утилит (`curl`, `git`, `htop` и др.) и полная настройка файрвола (UFW) под используемые сервисы.
* **Role: vpn_dns** — Развертывание VPN-панели **3X-UI** и DNS-сервера **Pi-Hole v6** с автоматическим импортом уже существующих баз данных и конфигураций.

## Требования (Prerequisites)
* На управляющем ПК должен быть установлен **Ansible 2.13+**.
* Настроен SSH-доступ по ключу к целевому серверу для пользователя с правами `sudo`.

## Быстрый старт

1. Склонируйте репозиторий:
   ```bash
   git clone <ссылка_на_ваш_репозиторий>
   cd vps-infrastructure
   ```

2. Создайте файл инвентаря `hosts` на основе примера:
   ```bash
   cp hosts.example hosts
   ```
   Отредактируйте `hosts`, указав IP-адрес вашего сервера и имя SSH-пользователя.

3. Поместите ваши существующие базы данных конфигураций в папку ролей:
   * Базу данных 3X-UI в: `roles/vpn_dns/files/x-ui.db`
   * Конфигурацию Pi-Hole v6 в: `roles/vpn_dns/files/pihole.toml`
   * Базу гравитации Pi-Hole в: `roles/vpn_dns/files/gravity.db`

4. Запустите развертывание плейбука:
   ```bash
   ansible-playbook playbook.yml -K
   ```
